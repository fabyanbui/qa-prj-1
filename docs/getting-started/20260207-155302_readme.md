# ShopPy - E-commerce QA Playground

ShopPy is a Next.js App Router e-commerce demo used for QA, automation, and full-stack practice.
It includes auth flows, role-based seller features, cart + checkout, Prisma + SQLite persistence, API routes, and both unit/component + E2E tests.

## Prerequisites

- Node.js 20+
- npm 10+

Optional (for first-time E2E setup):

- Playwright browser binaries via `npx playwright install --with-deps`

## Quick Setup

1) Install dependencies:

```bash
npm install
```

2) Create `.env` in the project root:

```bash
DATABASE_URL="file:./dev.db"
```

3) Generate Prisma client and seed sample data:

```bash
npx prisma generate
npx prisma db seed
```

4) Run the app:

```bash
npm run dev
```

5) Open `http://localhost:3000`.

## Dev + QA Commands

```bash
# Install Playwright browsers (first run only)
npx playwright install --with-deps

# Lint
npm run lint

# Unit/component tests (Vitest)
npx vitest

# E2E tests (Playwright)
npx playwright test
npx playwright test e2e/auth.spec.ts
```

Baseline note:

- Current repository state includes pre-existing lint/test failures unrelated to setup.
- If `npm run lint` or `npx vitest` fails immediately, treat it as current baseline, not environment misconfiguration.

## API Surface

- `POST /api/auth/signup`
- `POST /api/auth/login`
- `GET, POST /api/products`
- `GET, PUT, DELETE /api/products/:id`
- `POST /api/checkout`

## Important Notes

- Auth is demo-style (mock JWT token and plaintext password checks).
- Sessions are persisted in localStorage (`auth_sessions`, `active_session_email`).
- Cart state is persisted in localStorage (`cart`).
- Checkout groups cart items by `sellerId` and creates one order per seller.
